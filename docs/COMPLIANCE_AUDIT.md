### 2.1 Data Layer (AV 1.x)

| Rule | Description | Status | Notes |

| 1.1.1 | Never import from presentation layer | **PASS** | Verified: no presentation imports in any data layer file. |
| 1.1.2 | Do not import use_cases | **PASS** | Verified: no use_case imports in data layer. |
| 1.2.1 | Throw exceptions from data_sources | **PASS** | Data sources throw on failure; repository catches and wraps in `AppException`. `FirebaseAuthDataSourceImpl` catches `FirebaseAuthException` internally and throws `AppException` with user-friendly messages. |

This is the DI container (not a clean layer), so arguably exempt — but a strict reading could flag it. See [Flagged Item #7](#flagged-items). |
| 1.3.1 | Models ALWAYS extend an entity | **PASS** | `FirebaseArticleModel extends FirebaseArticleEntity`. `ArticleModel extends ArticleEntity`. `UserModel extends UserEntity`. |

| 1.4.1 | Repository implementations named `{Interface}Impl` | **PASS** | `CreateArticleRepositoryImpl`, `ArticleRepositoryImpl`, `AuthRepositoryImpl`. |
| 1.4.2 | Only repository implementations of domain abstract classes | **PASS** | All repo impls implement their corresponding domain interfaces. |
